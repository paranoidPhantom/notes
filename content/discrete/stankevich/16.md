# Теория информации

**Claude Shannon** дал такое определение информации:
$\text{инф} = -\text{неопределённость}$
$I=-\Delta H$
$I = H_1 - H_2$
$\ra \Delta H = H_2 - H_1$

$p_1, p_2, \dots, p_n \in \Omega$

## Случайный источник

Концепция неограниченного источника "экспериментов"; тоже вероятностное пространство.

## Энтропия Шеннона

**Единственная формула** для количества информации.

---

Пусть $p_1,p_2,\dots,p_n$ - случайный источник
$H(p_1,p_2,\dots,p_n): \text{RS} \rightarrow \mb R^+$ - энтропия

($\text{RS}$ - random sources, все корректные распределения вероятности в случайных источниках)

Рассмотрим частный случай для $p_i=\frac 1n$
Определим для него $h(n) = H\left(\frac 1n,\frac 1n,\dots,\frac 1n\right)$

*Тогда свойства энтропии Шеннона:*
1) $h(n+1) > h(n)$ - монотонность энтропии
2) Рассмотрим вер. пр-во:
	$\Omega=\{(1,1), (1,2), \dots, (1,m_1),$
	    $(2,1), (2,2), \dots, (2,m_2),$
	    $\qquad \qquad \qquad \vdots$
	    $(k,1), (k,2), \dots, (1,m_k)\}$
	Тогда число элементарных исходов $n = m_1 + m_2 +\dots+ m_k$
	$p(i,j) = q_{ij}$
	$p_i = \argsum {m_i}{j=1}q_{ij} \ra \argsum k {i=1} p_i = 1$
	
	Пусть у случайного источника "сломался дисплей" и он выдаёт только первый компонент пары. Тогда этот "сломанный" случайный источник эквивалентен случайному источнику с распределением вероятности $p_1,p_2,\dots,p_k$.
	Количество информации - $H(p_1,\dots,p_k)$
	
	Теперь "открываем" вторую компоненту.
	Тогда количество информации полученное на втором шаге:
		$\argsum k {i=1} p_i \cdot H\left(\frac {q_{i1}} {p_i}, \frac {q_{i2}} {p_i},\dots,\frac {q_{im_i}} {p_i}\right)$
	
	Теперь проведём эксперимент "разом", получив
		$H(q_11,\dots,q_k{m_k}) = H(p_1,\dots,p_k) + \argsum k {i=1} p_i \cdot H\left(\frac {q_{i1}} {p_i}, \frac {q_{i2}} {p_i},\dots,\frac {q_{im_i}} {p_i}\right)$ информации
	Это называется аддитивностью энтропии.
1) Для фиксированного $n$, $H$ - непрерывная как функция $\mb R^n \rightarrow \mb R$

### Теорема
 
$\exists! H(p_1,p_2,\dots,p_n)=-\alpha \argsum n {i=1}p_i\;log_2\;p_i \quad \alpha \in \mb R$
### Лемма 1

$h(nm) = h(n) + h(m)$
#### Доказательство
$k=n \quad m_i = m$
$p_i = \frac 1n \quad q_{ij} = \frac 1{nm} \quad r_{ij} = \frac 1m$
$h(nm) = H(q_{11},q_{12},\dots,q_{nm}) = H(\frac 1n,\frac 1n,\dots,\frac 1n) + \argsum n {i=1}\frac 1n H\left(\frac 1m,\dots,\frac 1m\right) = h(n) + h(m)$

Зафиксируем константу $\alpha \la h(2) = 1$

### Лемма 2

$h(2^k) = k\alpha \la$ *Лемма 1*

### Лемма 3

$h(n) = \alpha\;log_2\;n$

Рассмотрим $2^i \leq n \cdot r < 2^{i+1}$
$h(n^r) = rh(n)$
$\alpha i \leq h(n^r) < \alpha(i + 1)$
$\alpha i \leq rh(n) < \alpha(i + 1)$

$i \leq r\;log_2\;n < i + 1$
$\alpha\frac ir \leq \alpha\;log_2\;n < \alpha \frac{i+1}r$
$\forall r: \left|h(n) - \alpha\;log_2\;n\right| \leq \frac \alpha r$ ❗$\forall \alpha \in R$
$\ra h(n) = \alpha\;log_2\;n$

---

Докажем Теорему для $p_i \in \mb Q$
$p_i = \frac{a_i}b$
$m_i = a_i$
$r_{ij} = \frac 1{a_i}$
$q_{ij} = \frac 1b$

$H(\frac 1b,\frac 1b,\dots,\frac 1b) = H(p_1,\dots,p_k) + \argsum k {i+1} p_i H\left(\frac 1 {a_i},\dots,\frac 1 {a_i}\right)$
$h(b) = H(p_1,\dots,p_k) + \argsum k {i+1}p_ih(a_i)$
$H(p_1,\dots,p_k)=\argsum k{i=1}p_i\left(\alpha\;log_2\;b-\alpha\;log_2\;a_i\right) = -\alpha\argsum k {i=1}p_i\;log_2\;p_i$

---

$h(2) = \alpha$ - **бит**

>Что если перенести количество информации со случайного источника на что-то детерминированное?

1) Это не корректный перенос
2) Если перенести на данные похожие на случайные, будет что-то $\pm$ содержательное

---

Пусть $S \in \Sigma^* \normalsize \quad n = \left | \Sigma \right |$
$|s| = L \quad f_i$ - кол-во $c_i$ в $s$
$p_i = \frac{f_i}L$

Проводим $L$ экспериментов и результаты записываем в строку: $\tilde s = s_1s_2\dots s_L$

$I(\tilde S) = L\cdot H(p_1,p_2,\dots,p_n) = -L\argsum n {i=1}p_i\;log_2\;p_i$

Вспомним арифметическое кодирование:

![screenshot](/_media/Pasted%20image%2020251103234002.png)

$q = A(s)$ - длина арифметического кодирования
$A(s) \leq -log_2(b_L - a_L) = -log_2\left(p_{s_1},p_{s_2},\dots,p_{s_L}\right) = -log_2\left(\arglarge \sqcap n {i=1}p_i^{f_i}\right)=$
$=-\argsum n{i=1}Lp_i\cdot log_2\;p_i = L \cdot H(p_1,\dots,p_n) = I(\tilde s)$
### $\ra$Теорема

Длина кода который получается для строки $s$ после арифметического кодирования не превышает энтропии Шеннона для случайного источника с частотами равными долям вхождения в строку.

$\ra$ *Хуже всего сжимаются случайные строки*
$\ra$ Арифметическое кодирование оптимально если можно учитывать только количество вхождений символов
